# Git Rules

> Simple. Fast. Don't overthink it.

- `main` stays deployable
- Feature branches: `feat/<short-name>`
- Conventional commits: `feat:`, `fix:`, `style:`, `refactor:`, `chore:`
- Pull main before merging: `git pull origin main`
- No PR reviews required — trust your teammates, move fast
- Don't commit `.env`, secrets, or `node_modules/`
- Check the ownership map before editing files in someone else's area
